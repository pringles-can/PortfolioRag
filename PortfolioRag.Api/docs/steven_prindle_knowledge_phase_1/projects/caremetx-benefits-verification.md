## Technical Problem: EDI Parsing Defect

An in-house Java rule engine was stripping segments that indicated active Medicare Supplement plans. Those records were written to the database after mutation of a Java array where the plan letter's index was explicitly modified.

Records were then read from the database using a query generated by the Hibernate Criteria API. An off-by-one error caused the plan letter to be truncated, which led business logic to invalidate Medicare Supplement plans that should have been valid.
