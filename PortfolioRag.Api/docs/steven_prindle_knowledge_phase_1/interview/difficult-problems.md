# Interview Answer — Difficult Technical Problems

## EDI Parsing Defect

At CareMetX, Steven diagnosed an improper EDI parsing issue involving a Java rule engine, Hibernate Criteria API queries, and an off-by-one truncation bug. The issue caused Medicare Supplement plans that should have been valid to be invalidated by business logic.

## Performance Timeout

At CareMetX Data Egress & Transformation, Steven identified a solution to a work-stopping timeout, leading to 80% faster load times on common pages.

## LINQ Performance

Steven improved page response time by 66% by eliminating premature `.ToList()` calls, reducing multiple enumeration, precompiling expensive SQL-generating LINQ functions, and refactoring expressions to avoid broad wildcard string matching.

## Batch Runtime Modernization

At MDHHS, Steven helped migrate legacy Oracle batch billing jobs to Groovy scripts, reducing runtime from 10–12 hours to about 30 minutes.
